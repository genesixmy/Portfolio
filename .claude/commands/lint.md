# Run ESLint

Check code quality and find potential issues.

This command runs ESLint on the entire codebase to find style issues, potential bugs, and violations of best practices.

**Usage:**
```bash
npm run lint
```

**What to look for:**
- Style inconsistencies
- Unused variables
- Potential bugs
- Type errors
- Best practice violations

**Fixing issues:**
- Many issues can be auto-fixed with `npm run lint -- --fix`
